# 用户信息管理平台

一个基于 **Python Flask** 构建的简易用户信息管理平台，提供用户登录认证、信息展示、搜索、头像上传、个人中心、充值、密码修改、动态页面加载等功能。该项目专注于**安全性最佳实践**，适合作为 Flask 入门学习和安全编码参考。

---

## 功能特性

- ✅ **用户登录认证** — 基于 Session 的登录态管理
- ✅ **用户注册** — 新用户可自行注册账号
- ✅ **用户搜索** — 登录后可按用户名或邮箱模糊搜索用户
- ✅ **用户信息展示** — 登录后可查看个人完整信息
- ✅ **头像上传** — 登录后可上传头像图片，支持即时预览
- ✅ **个人中心** — 登录后查看个人详细资料（邮箱、手机、余额）
- ✅ **账户充值** — 支持余额充值，实时更新账户金额
- ✅ **密码修改** — 支持修改登录密码，需验证原密码
- ✅ **动态页面加载** — 通过 URL 参数加载 pages/ 目录下的 HTML 页面
- ✅ **安全登出** — 清除 Session 安全退出
- ✅ **密码哈希存储** — 使用 Werkzeug 安全哈希，不存明文密码
- ✅ **CSRF 防护** — 所有 POST 表单携带一次性 Token，防止跨站请求伪造
- ✅ **SQL 注入防护** — 所有数据库查询使用参数化查询（? 占位符）
- ✅ **SQLite 数据库** — 注册、搜索、充值数据持久化存储
- ✅ **响应式界面** — 简洁现代的 UI 设计，适配多种屏幕

## 快速开始


## 头像上传功能

登录后可通过以下方式进入上传页面：

- **导航栏**：点击右上角"上传头像"链接
- **快捷入口**：首页欢迎页中点击"上传头像"按钮
- **直接访问**：打开 `http://localhost:5000/upload`

### 上传说明

- 仅允许上传图片格式文件（jpg、jpeg、png、gif、webp、bmp）
- 上传采用用户原始文件名保存
- 上传成功后页面会显示图片预览和文件访问链接
- 上传文件大小限制：**16 MB**
- 文件保存在 `static/uploads/` 目录下
- 未登录用户访问 `/upload` 会自动跳转到登录页

## 个人中心、充值、密码修改

登录后可通过以下方式进入个人中心：

- **导航栏**：点击右上角"个人中心"链接
- **快捷入口**：首页欢迎页中点击"个人中心"按钮
- **直接访问**：打开 `http://localhost:5000/profile`

### 功能说明

**用户资料展示**
- 显示当前登录用户的详细资料（ID、用户名、邮箱、手机、余额）
- 用户仅可查看自己的账户信息，无法查看其他用户资料

**账户充值**
- 输入金额即可为当前账户充值
- 充值金额必须为正数，金额实时更新到数据库
- 用户只能给自己充值

**密码修改**
- 输入原密码验证身份，再输入新密码完成修改
- 需要原密码正确才能修改，防止未授权密码更改
- 需携带 CSRF Token，防止跨站请求伪造
- 用户只能修改自己的密码

## 动态页面加载功能

通过 URL 参数加载 pages/ 目录下的 HTML 页面内容，并嵌入首页显示。

### 使用方式

- **快捷入口**：首页点击"帮助中心"按钮
- **直接访问**：`http://localhost:5000/page?name=help`

### 功能说明

- 从 `pages/` 目录读取 HTML 文件，将内容嵌入首页显示
- 支持带 `.html` 后缀（`name=help.html`）或不带后缀（`name=help`）访问
- 文件不存在时显示提示信息
- 已对路径穿越字符（`../`、`..\`）进行过滤，防止文件包含漏洞

## 安全设计

本项目在开发过程中注重安全性，实施了以下防护措施：

### 1. 密码安全

- **哈希存储**：密码使用 `werkzeug.security.generate_password_hash()` 进行 PBKDF2 哈希
- **原密码验证**：修改密码需要验证原密码，防止未授权更改
- **安全比对**：登录验证使用 `check_password_hash()`，防止时序攻击

### 2. CSRF 防护

- 所有 POST 表单（登录、充值、密码修改）生成并携带一次性 `csrf_token`
- Token 存储在服务端 Session 中，每次 POST 请求校验一致性
- 防止跨站请求伪造攻击

### 3. 敏感信息保护

- 用户密码**不在前端页面展示**
- 非敏感信息（用户名、邮箱、手机号、角色、余额）正常展示

### 4. 调试安全

- 生产环境关闭 `debug` 模式，防止调试器远程代码执行

### 5. XSS 防护

- Jinja2 模板引擎默认开启自动转义，所有 `{{ }}` 变量中的 HTML 特殊字符会被转义，防止跨站脚本攻击

### 6. SQL 注入防护

- 所有数据库查询使用**参数化查询**（`?` 占位符）
- 用户输入作为数据传入，不会被解析为 SQL 代码
- 注册、搜索、充值、密码修改功能均经过安全加固

### 7. 文件上传安全

- 上传文件后缀名白名单校验，仅允许图片格式
- 限制了全局上传文件大小为 16 MB

### 8. 越权访问防护

- 个人中心、充值、密码修改功能基于 Session 识别用户身份，不依赖客户端传入的 user_id 参数
- 用户只能查看和操作自己的账户，无法访问其他用户的资料或修改其他用户的密码

### 9. 充值金额校验

- 充值接口对金额进行正负校验，拒绝非正数金额
- 非数字输入被自动拦截

### 10. 文件包含防护

- 动态页面加载接口对路径穿越字符（`../`、`..\`）进行过滤
- 防止攻击者通过路径穿越读取 pages/ 目录外的任意文件

## API 接口一览

| 路由 | 方法 | 说明 |
|------|------|------|
| `/` | GET | 首页，已登录显示用户信息，未登录提示登录 |
| `/login` | GET | 显示登录表单 |
| `/login` | POST | 提交用户名和密码进行登录 |
| `/register` | GET | 显示注册表单 |
| `/register` | POST | 提交用户名、密码、邮箱、手机号进行注册 |
| `/search` | GET | 按用户名或邮箱模糊搜索用户（需登录） |
| `/upload` | GET | 显示头像上传页面（需登录） |
| `/upload` | POST | 接收上传文件，返回图片预览和文件链接（需登录） |
| `/profile` | GET | 显示当前登录用户的个人中心、充值、密码修改页面（需登录） |
| `/recharge` | POST | 处理充值请求，更新账户余额（需登录） |
| `/change-password` | POST | 验证原密码后修改密码（需登录） |
| `/page` | GET | 加载 pages/ 目录下的 HTML 页面内容并嵌入首页显示 |
| `/logout` | GET | 清除 Session 并重定向到首页 |

## 自定义扩展

### 添加新用户

**方式一**：在 `app.py` 的 `USERS` 字典中添加新条目（登录可用）：

```python
USERS = {
    # ... 已有用户 ...
    "bob": {
        "username": "bob",
        "password": generate_password_hash("bob2025"),
        "role": "user",
        "email": "bob@example.com",
        "phone": "13700137001",
        "balance": 500
    }
}
```

**方式二**：通过网页 `/register` 注册（写入 SQLite 数据库）

### 修改监听端口

```python
app.run(debug=False, host="0.0.0.0", port=8080)  # 改为 8080 端口
```

### 修改上传文件大小限制

```python
app.config["MAX_CONTENT_LENGTH"] = 32 * 1024 * 1024  # 改为 32 MB
```

### 添加动态页面

在 `pages/` 目录下创建新的 HTML 文件，通过 `/page?name=文件名` 即可加载。
